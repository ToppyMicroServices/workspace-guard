# Governance

Workspace Guard is maintained by ToppyMicroServices.

## Decision Making

- Maintainers are responsible for release decisions and repository policy.
- Security-sensitive changes should prefer conservative defaults and explicit review.
- Breaking changes should be documented in the pull request and release notes.

## Repository Policy

- `main` is protected.
- Changes land through pull requests.
- Required checks must pass before merge.

## Releases

- Stable releases are published from GitHub releases.
- Daily builds are published as prereleases.
